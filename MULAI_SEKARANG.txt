Q: Bagaimana kalau salah input?
A: Klik tombol "HAPUS" pada pengeluaran yang salah

Q: Bisa input riwayat lama?
A: Ya! Ubah tanggalnya ke tanggal yang lebih tua

Q: Data aman tidak?
A: 100% aman! Tersimpan lokal di komputer, tidak dikirim ke server

Q: Bisa lihat bulan kemarin?
A: Ya! Gunakan filter "Filter Bulan" di section Detail Pengeluaran

Q: Data bisa hilang tidak?
A: Hanya kalau Anda clear cache browser atau reset data

Q: Perlu install aplikasi?
A: Tidak! Buka di browser, langsung jalan

Q: Perlu internet?
A: Tidak! 100% offline, data disimpan di komputer

║                    💡 TIPS PENGGUNAAN OPTIMAL                   ║
╚══════════════════════════════════════════════════════════════════╝

TIPS #1: Input Konsisten
───────────────────────
Lebih baik 1 input per hari dari pada 30 input di akhir bulan
→ Lebih mudah ingat
→ Lebih akurat
→ Lebih mudah tracking

TIPS #2: Deskripsi Jelas
───────────────────────
Jangan: "Makan", "Belanja", "Kendaraan"
Lebih baik: "Sarapan nasi kucing", "Belanja di Indomaret", "Bensin motor"
→ Easier recall
→ Better categorization
→ More meaningful insights

TIPS #3: Kategori Konsisten
───────────────────────────
Pilih kategori yang paling match, jangan tebakan
Jika ragu, gunakan "Lainnya"
→ Data lebih reliable
→ Grafik lebih akurat

TIPS #4: Review Weekly
─────────────────────
Setiap minggu, ambil 5 menit untuk:
- Review pengeluaran minggu lalu
- Lihat kategori mana yang paling boros
- Plan pengeluaran minggu depan
→ Lebih aware & mindful

TIPS #5: Backup Data
────────────────────
Setiap akhir bulan:
1. Buka F12 (DevTools)
2. Buka Console tab
3. Copy-paste hasil: JSON.stringify(JSON.parse(localStorage.getItem('expenses')), null, 2)
4. Save ke file .json
→ Data terbackup & bisa restore
